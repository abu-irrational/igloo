DESCRIPTION

Igloo(n) is a layer on top of Tcl's oo:: object system that provides Snit-like convenience. Igloo provides its own class building tools, igloo::class create and igloo::define, that provide access to the following capabilities:

  • Instance variable declarations with initial values
  • Snit-like option handling
  • Snit-like delegation

Classes defined with igloo(n) are normal oo:: classes; an igloo::class can subclass and be subclassed by any normal oo:: class.

GLOBAL COMMANDS

Igloo(n) provides the following global commands for creating and configuring classes:

igloo::class create class ?defScript?
Defines a new class called class in the caller's context, returning the fully-qualifed class name. If defScript is given, it is an igloo::define script.

igloo::define class defScript
igloo::define class subcommand args...
Adds one or more definitions to an existing class, which must name a class in the caller's context.

If defScript is given, it should be a script of zero or more of the CLASS DEFINITION COMMANDS; otherwise, subcommand should be a single definition command with its args.

CLASS DEFINITION COMMANDS

The following commands may appear in igloo::class create and igloo::define defScripts, and as subcommands of igloo::define:

NOTE: We need to test that these commands can be used at any time, and will affect existing objects in the logical way. If the command is marked "As for oo::define", then the command's semantics are essentially the same as for oo::define, though possibly with some additional magic behavior.

constructor arglist body
As for oo::define. Defines the class constructor, where arglist is the constructor's argument list and body is the constructor's body. If the class defines options, the arglist should usually be the single token "args".

If no constructor is define, and options are defined, the class will have a default constructor that handles creation options.

method name arglist body
As for oo::define. Defines an instance method in the usual way.

option name TBD
Declares an option; the syntax is TBD but should match Snit, at least in capability. Classes that define options will have cget and configure methods. If no explicit constructor is given, then a default constructor that handles creation options will be provided.

If the class does provide a constructor, it is the constructor's responsibility to pass the creation options to configure. NOTE: when gluing components together in the presence of option delegation, it is often necessary to create the components before options are handled. This rule allows the programmer to control the order of events as needed.

superclass name arglist body
As for oo::define. Specifies one or more superclasses for the class. Note that Igloo classes can subclass both igloo:: oo:: classes.

variable name ?-array? ?initValue?
Declares an instance variable called name, making it visible in all method bodies. If the initValue is given, the variable will be initialized with this value before the constructor is called; otherwise it will be set to the empty string.

If the -array option is given, then the variable is an array variable, and the initValue will be assigned to the variable using array set.

INSTANCE COMMANDS

Instances of igloo classes may have the following standard subcommands:

object cget option
Returns the value of the option.

object configure option value ?option value...?
Sets the values of one or more options.
